# 调试会话: 货币符号和数据显示问题

## 📋 会话信息
- **会话ID**: currency-symbol-bugs
- **状态**: [OPEN]
- **开始时间**: 2026-06-23
- **问题描述**:
  1. 图表页面金十数据显示 $ 而不是 ¥
  2. 历史数据页面显示"暂无行情记录"
  3. 首页数据源切换栏背景色错误（浅色主题显示深色）

---

## 🧪 假设列表

### 假设1: 数据源状态传递错误
**描述**: 数据源选择状态没有正确传递到图表页面的货币符号逻辑
**验证点**: 检查 `_selectedSource` 在图表页面初始化和切换时的值
**期望证据**: 日志显示 `_selectedSource` 值与预期不符

### 假设2: API数据解析问题
**描述**: 历史数据API返回的数据格式与预期不一致
**验证点**: 检查 `getMarketHistory` 返回的数据结构
**期望证据**: 日志显示数据为空或格式错误

### 假设3: 主题状态读取错误
**描述**: 首页数据源切换栏没有正确读取当前主题状态
**验证点**: 检查 `Theme.of(context).brightness` 的返回值
**期望证据**: 日志显示 brightness 值与实际主题不符

### 假设4: 数据源标识符不匹配
**描述**: 使用了不一致的数据源标识符（如 'jinshi' vs '金十数据'）
**验证点**: 检查所有数据源判断逻辑中的标识符
**期望证据**: 日志显示不同位置使用了不同的标识符格式
